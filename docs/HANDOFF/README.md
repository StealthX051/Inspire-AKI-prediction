# Handoff Notes

Use this folder for explicit session handoffs between people working on the repo.
Do not create a handoff for a read-only check-in that does not materially change the repo state or the next recommended step.

Suggested filename pattern:

- `YYYY-MM-DD_short-topic.md`

Good handoffs usually include:

- what changed
- what commands ran
- what artifacts or outputs matter
- current blockers or risks
- the next recommended command or next concrete step

Prefer short dated notes with links to code, config, and artifact paths instead of pasting long logs.
Prefer updating an existing same-day handoff for the same workstream over adding a second near-duplicate note.
If a note is superseded and no longer adds unique context, delete it rather than letting the folder accumulate stale summaries.
