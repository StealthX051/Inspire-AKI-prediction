## Current CLI Manuscript Contract

- `report manuscript` is the top-level manuscript export command.
- Manuscript-facing tables are emitted as `html`, `md`, and `csv`.
- Manuscript-facing figures are emitted as `png` and `svg`.
- The maintained report layer supports consort, cohort tables, calibrated and uncalibrated performance tables, bootstrap CI tables, raw and FDR-corrected DeLong tables, DCA outputs, reclassification summaries, and SHAP outputs when configured.
- The maintained SHAP surface now emits importance CSVs, beeswarms, raw-value single-feature scatter plots, and opt-in manual dependence plots from the CLI rather than requiring notebook-local reconstruction.
- The maintained scatter and manual dependence plots now cover the full held-out split rather than a small explained-row sample, which restores notebook-style point coverage while keeping the explainer background matrix bounded.
- The default AKI config marks `BSA`, `max_hr`, `max_inibg_mbp`, `op_len`, `preop_hb`, and `ward_rr` as the featured manuscript scatter set and mirrors those figures under `reports/figures/shap_scatter_featured/` when available.
- AKI compatibility aliases still exist through `compat export-legacy`, but they are explicit and AKI-only.
- Current shipped configs default to patient-grouped evaluation modes rather than the historical operation-level repeated-CV path.
- When stages are run manually, `evaluate generate` is the maintained entrypoint that materializes those patient-grouped manifests before tuning and training.
- Calibration in the maintained CLI is grouped on `op_id`, so repeated predictions from the same operation do not leak across isotonic calibration folds.
- The default AKI config now evaluates `asa_rule` and `gs_aki_rule` on the same maintained grouped manifests used by the learned models.
- `gs_aki_rule` is implemented as one adapted preoperative clinical baseline only; the current code does not refit or recalibrate a separate GS-AKI family of models.
- In manuscript-facing tables, `gs_aki_rule` should be treated primarily as an ordinal count/class score. Its threshold-dependent columns are intentionally blanked in the main performance table, while `asa_rule` remains a prespecified binary rule.
- `report procedure-audit` is available as an explicit audit stage. It audits final analytic-cohort `op_id` rows against raw `operations.csv`, uses `icd10_pcs` as the primary classifier, and treats `cpbon_time` / `cpboff_time` as secondary CPB sanity fields.
- The maintained reviewer-response bundle now also includes the focused scripts `scripts/department_os_audit.py` and `scripts/department_ot_reviewer_report.py` for narrow department provenance and ophthalmology subgroup questions; these scripts are outside the CLI stage map and write repo-local `reports/` outputs by default.
- The maintained reviewer-response bundle also includes `scripts/run_reviewer_missingness_sensitivity.sh` plus `scripts/combined_xgb_missingness_sensitivity.py` for the combined `xgb` missingness sensitivity analysis; these are separate reviewer utilities rather than new default CLI stages because the alternative preprocessing must stay fold-fit on the grouped outer manifests.
- The maintained audit now auto-retains the clearly noncardiac CTS service families, narrows the residual clinician-review set into explicit buckets, and records a final keep-versus-exclude recommendation for each bucket.
- The current mounted raw-data contract used by the maintained audit stores 5-character `icd10_pcs`.
- The shipped default AKI config now points at the strict operation-level adjudicated noncardiac cohort. `configs/aki/strict_noncardiac_review.yaml` remains only as a compatibility alias to rerun that same cohort into a separate artifact root.
- Until the next full rerun, existing default artifact trees should be treated as stale relative to this cohort-default promotion and should not be cited for updated counts.
- The cohort characteristics table still reports total operations from unique `op_id`, but its department rows are patient-level counts after deduplicating `subject_id`; that aggregation should not be conflated with operation-level department features such as `department_OS`.
- The cohort characteristics table now uses `Department, n (%)` so the manuscript-facing table no longer presents department as a surgery-type label.
